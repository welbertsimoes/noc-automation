Automatizei um fluxo de NOC integrando Zabbix + n8n + GLPI + Microsoft Teams.

A ideia foi reduzir trabalho manual no tratamento de alertas e evitar chamados duplicados.

O workflow principal recebe eventos do Zabbix via webhook autenticado, valida o payload, classifica o alerta, consulta histórico, verifica chamados abertos no GLPI e decide automaticamente o melhor caminho:

→ abrir novo chamado
→ atualizar chamado existente
→ solicitar validação humana no Teams
→ aguardar próximo step de escalonamento

Pontos técnicos que implementei:

✅ Webhook autenticado
✅ Validação de payload
✅ Classificação de eventos
✅ Consulta de histórico no Zabbix
✅ Detecção de recorrência/flapping
✅ Deduplicação antes da abertura de ticket
✅ Integração com GLPI
✅ Card interativo no Microsoft Teams
✅ Acknowledge automático no Zabbix
✅ Estratégia anti-colisão para eventos simultâneos
✅ Tratamento de timeout na validação humana

Publiquei no GitHub uma versão de showcase, com arquitetura, documentação e amostras sanitizadas do workflow — sem expor código operacional completo, URLs internas, tokens ou regras sensíveis.
